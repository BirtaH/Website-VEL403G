Once I was satisfied with all of the content the next step is to upload everything to Github. I choose to use Bit Bash, following the instructors instructions and of course to include version control. Detailed instructions on how I uploaded my website are here below:

1. I start by creating a repository on Github and naming it “Website-VEL403G”

2. To open Git Bash I right click somewhere on the desktop and choose Git Bash and navigate to the websites folder using the command cd (Location_of_website). Or right click inside the folder where the websites files are located and choose Git Bash.

3. Start by setting your username, using git config --global user.name "FIRST_NAME LAST_NAME".

4. Then set your email address, using git config --global user.email MY_NAME@example.com.

5. Type git init in the command line to initialize the local directory as a Git repository.

6. Type git add . in the command line to stage the files for the first commit and add them to the local repository.

7. Type Git commit -m “appropriate text” in the command line to commit the changes made in your local repository.

8. Type git remote add origin repository_URL to choose a repository on Github where your local repository will be pushed.

9. Type git push origin master to push the changes in your local repository to the Github repository.

10. Sign in to your Github account and your local repository has been pushed.

To save changes to the local repository in the Github repository follow these steps:

1. Type git add . in the command line to stage the files for the first commit and add them to the local repository.

2. Type Git commit -m “appropriate text” in the command line to commit the changes made in your local repository.

3. Type git push origin master to push the changes in your local repository to the Github repository.
